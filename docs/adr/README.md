# ADR

ADR은 방향·구조·철학만 담는다. 구현 디테일은 코드가, 변하는 외부 사실(가격 등)은 `docs/vendors/`가 진실이다.

## 용어

| 용어 | 뜻 |
|---|---|
| **클라이언트 (client)** | filegate에서 lease를 받아가는 서비스의 등록 단위 (notegate 등). 설정의 `clients` 블록 하나 = 서비스 하나 |
| **프로바이더 (provider)** | 등록된 저장 공간 접근 계약: 벤더 + 계정 + 주소 + 자격증명 + 공간. 같은 벤더의 다른 계정도 다른 provider |
| **intent** | 클라이언트가 파일 용도를 말하는 어휘 (`note_attachment` 등). 클라이언트별 네임스페이스 |
| **profile** | 운영자가 정의한 배치·수명 정책 템플릿. intent가 참조 |
| **file / location** | 파일의 정체성(불변 id) / 물리 위치(가변 포인터). 분리가 이동을 가능케 함 |
| **lease** | 시간제한·취소가능·단일목적 접근 권한. 모든 바이트 접근의 단위이자 감사 단위 |
| **quota** | 클라이언트에게 약속한 용량 몫 |
| **capacity** | provider의 물리 한도 (무료 구간, 디스크) |
| **reconciler** | 요청 경로 밖에서 물리 상태를 정리하는 백그라운드 작업 (GC, purge, 이동) |
