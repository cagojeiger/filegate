# ADR 002: 모든 바이트 접근은 lease다

- Status: Draft
- Date: 2026-07-03
- 부모: [000](000-identity.md) 공리 2 (바이트 직결이 기본)

## 문제

filegate는 바이트를 직접 보지 않는다. 그래도 접근 발급, 기록, 검증은 해야 한다. S3 facade는 쓰지 않는다. 바이트 프록시를 강제하고 intent를 표현하기 어렵기 때문이다. 서명 URL만 쓰면 크기 강제와 전송 완료 확인이 어렵다.

## 결정

- **lease는 단일목적 접근 권한이다.** 시간제한과 취소 가능성을 가진다. 쓰기, 읽기, 사용자 위임 모두 lease를 통한다.
- **쓰기는 발급 → 확정 2단계다.** 발급 시 capacity를 예약한다(quota 축은 방향이며 이번 범위 밖 — ADR 004). 전송 주체는 저장소에 직접 업로드한다. commit 시 filegate가 크기와 완결성을 검증하고 정산한다.
- **commit이 검증 게이트다.** commit되지 않은 바이트는 파일이 아니며, 만료 후 회수된다.
- **읽기는 매번 현재 location을 해석한다.** 파일이 이동해도 같은 file_id로 접근한다.
- **접근 수단은 capability로 결정한다.** 기본은 저장소가 발급한 presigned URL이다. presigned 또는 CORS가 불가능하면 filegate 중계 URL을 발급한다.
- **직결과 중계는 서비스에 같은 계약이다.** 발급, 전송, 확정으로 동일하다.
- **중계는 filegate에 둔다.** 서비스마다 fallback을 구현하지 않는다.
- **중계 토큰은 lease별 secret으로 검증한다.** 객체, 동작, 만료 바인딩은 lease가 담당한다.
- **lease가 감사 단위다.** 접근 추적, 고아 탐지, 감사 기록은 lease에서 나온다. 생애는 발급 → 확정 | 만료 | 취소다.
- **보안 검사는 발급 전에 한다.** 서비스는 유저 권한을 검사하고, filegate는 클라이언트를 검사한다. TTL 값은 설정으로 둔다.

모드별 성질:

| | 직결 (암호서명형) | 중계 (서버기록형) |
|---|---|---|
| 검증 | 저장소가 서명 산술로 | filegate가 lease 조회로 |
| 발급된 URL 회수 | 불가 (만료까지 유효) | 즉시 가능 |
| 실사용 관측 | 불가 | 가능 (시각·전송량) |
| 크기 강제 | 불가 → 확정 게이트 사후 검증 | 스트림 중 즉시 차단 |

## 흐름

업로드의 단계별 시퀀스(발급 → 직접 PUT → 확정)는 [spec 00](../spec/00-operations.md)에 둔다. 서비스가 직접 PUT하는 서버측 업로드도 같은 흐름이다.

## 경계선

- 직결 모드는 사용 시점을 알 수 없고 발급된 URL을 회수할 수 없다. 짧은 TTL과 commit 검증으로 다룬다. lease 취소는 원장 기준이다. 취소된 lease로는 확정할 수 없다.
- 브라우저 위임 쓰기는 저장소 CORS가 필요하다. CORS 미지원이면 중계 모드를 쓴다.
- 중계 URL의 만료는 lease 만료를 따른다.
- 읽기는 용량을 소비하지 않는다. 용량 = 물리적 점유(purge 전까지).
- 대용량은 multipart로 다룬다 — write lease가 URL 하나 대신 part 집합을
  가진다 (필요가 확인되어 확정, 2026-07-11). 갱신 = 필요한 part 접근의
  재발급이며, 이것이 재개다. 재개는 part 단위다 — part 내부 오프셋 재개는
  하지 않고, part 크기가 재전송 비용의 상한이다.
- multipart의 검증 단위는 part 실측이다. 벤더의 multipart ETag는 전체
  MD5가 아니므로, 전체 MD5는 선언 시 검증하는 선택 항목으로 내려간다.
  part 크기·발급 경로·정리 등 세부 계약은 spec에 둔다.

## 결과

- 클라이언트 컨트롤 API 표면은 발급, 확정, 파일 조회, 삭제 결정으로 유지한다. 용량 조회(usage)는 클라이언트가 아니라 운영자 표면이다(ADR 003). 중계 모드의 바이트 엔드포인트는 컨트롤 표면이 아니라 데이터 표면이다.
- 요청 경로는 정책, 배치, 발급, 기록만 한다. 회수, 검증, 보존 집행은 reconciler가 처리한다.
- filegate 장애에도 발급된 접근과 진행 중 전송은 계속된다.
- 서비스는 접근 수단을 불투명하게 취급한다. URL 구조 의존은 계약 위반.
