# ADR 006: 백엔드는 Rust다

- Status: Draft
- Date: 2026-07-08
- 부모: [000](000-identity.md) 공리 3 (내부 의존 0), [002](002-lease-model.md) 중계 스트리밍, [005](005-metadata-store.md) 단일 바이너리 배포

## 문제

filegate는 두 성격의 부하를 한 프로세스에서 처리한다: 저지연 컨트롤 플레인(발급·확정·회계)과, 중계 모드의 바이트 스트리밍 패스스루다. 배포는 바이너리 + PostgreSQL + 설정으로 끝나야 하고(공리 3, ADR 005), 런타임 의존을 늘리지 않아야 한다.

## 결정

- **백엔드 언어는 Rust다.** 컨트롤 플레인과 중계 데이터 플레인을 한 바이너리에 담는다.
- **중계는 버퍼링 없는 스트리밍으로 다룬다.** 선언 크기에서 스트림을 끊는 요구(ADR 002)를 GC 압박 없이 상수 메모리로 처리한다.
- **S3 호환 접근은 성숙한 S3 SDK로 한다.** provider adapter(ADR 001)의 1차 계약이 S3 호환이므로 검증된 클라이언트를 쓴다.
- **정적 링크 단일 바이너리를 배포 산출물로 삼는다.** 런타임·인터프리터 의존이 없다.

## 경계선

- 프레임워크·크레이트 선택은 구현의 영역이다. 이 ADR은 언어와 배포 형태만 고정한다.
- 성능이 이 선택의 유일한 이유는 아니다. 타입·소유권이 lease 상태 기계와 회계 불변식을 컴파일 타임에 강제하는 것이 같은 비중이다.

## 결과

- 배포 산출물 = 단일 실행 파일. 로컬은 docker-compose(MinIO + PG)로 띄운다.
- 삭제된 Rust 프로토타입에서 lease 상태 기계, commit 게이트, endpoint 분리, 회계 예약·정산을 이미 검증했다 — 이 선택은 실측에 기반한다.
