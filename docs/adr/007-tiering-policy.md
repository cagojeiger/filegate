# ADR 007: 파일 배치는 storage 소유 정책으로 cold에 수렴한다

- Status: Accepted
- Date: 2026-07-20
- 부모: [001](001-multi-storage.md) (file/location 분리 — 이동은 포인터 교체), [002](002-lease-model.md) (lease 원장 — 접근 관찰이 배치 신호가 된다), [004](004-config-layers.md) (정본은 DB, 선언은 Terraform)

## 문제

플릿은 성격이 다른 storage들로 구성된다: 무료 구간이 작은 대신 서빙이
좋은 객체 스토리지(R2·OCI)와, 용량·비용 한계가 사실상 없는 대신 홈
대역폭으로 서빙하는 NAS. 파일은 client의 기본 storage(hot)에서
태어나는데, 시간이 지나 읽히지 않는 파일이 작은 무료 구간을 차지하면
자주 읽히는 파일의 자리가 없다. 배치를 파일 단위로 수동 관리할 수는
없고, capacity를 집행(업로드 거부)하는 것은 관찰 공리에 어긋난다.

## 결정

> **배치는 정책이 다스린다.** 정책은 storage가 소유하는 등록부
> 리소스이며 — `(조건들의 AND, 목적지, 우선순위)` — 조건을 만족하는
> 파일의 **이동을 생성만** 한다. 집행은 이동 primitive(복사 → 검증 →
> 포인터 스왑 → 지연 삭제)가 하고, 파일은 정책 합성을 따라 cold로
> 수렴한다. 클라이언트는 이 전부를 모른다.

- **이동의 황금률**: dest 복사가 검증되고 스왑이 커밋되고 presigned
  수명만큼의 지연이 지나기 전에는 source 실물을 지우지 않는다. 어느
  시점에나 검증된 실물이 최소 하나 존재한다 — 이동 포기는 항상
  원위치로 안전하다.
- **경합은 요청 경로가 이긴다.** 이동 중 삭제·덮어쓰기가 오면 조건부
  전이가 이동을 조용히 패배시킨다. 정책이 아무리 틀려도 최악은
  "불필요한 이동"이지 손실이 아니다.
- **정책은 결정 생성기다.** 바이트·벤더 호출 없이 이동 저널에 INSERT만
  한다 (결정·집행 분리). 스케줄러(reconciler)가 우선순위 순으로 정책
  리스트를 순차 평가한다 — 앞 정책이 집은 파일은 빠지고 나머지가 다음
  정책의 입력이다 (첫 매칭 승리).
- **트리거는 둘이다**: 압력(사용량이 high 워터마크를 넘을 때만, low에서
  멈춤 — 히스테리시스)과 무조건(조건 매칭 즉시 — 대용량 파일의 cold
  직행). capacity는 여전히 집행하지 않는다 — 업로드는 초과해도 받고,
  정책이 빠르게 빼낸다 (과금이 용량-시간이라 짧은 초과는 비용이 없다).
- **방향 중립.** 정책은 출발지·목적지 쌍일 뿐이라 강등과 승격이 같은
  문법이다. 쿨다운(최근 이동 제외)이 양방향 정책의 핑퐁을 끊는다.
- **배치 신호는 이미 있는 관찰이다.** idle = lease_history의 마지막
  read (없으면 확정 시각) — 별도 카운터를 만들지 않는다 (공리:
  회계는 관찰).
- **실패는 층마다 기록된다**: 정책 평가 실패는 정책 행에, 개별 이동
  실패는 저널(재시도·park)에, 종결은 원장(move_history)에. 원장은 FK
  없이 박제되어 파일·storage가 정리된 뒤에도 홀로 읽힌다.
- **클라이언트 비가시는 구조가 보장한다.** 클라이언트의 계약은
  file_id뿐이고(ADR 003), 읽기는 location을 매번 재해석하며, stat은
  물리를 노출하지 않는다 (ADR 001). 티어는 운영자 세계에만 존재한다.

## 기각

- **용량 워터마크 단독 자동평형** — 객체 스토리지는 사실상 무한이라
  트리거되지 않는 죽은 정책이 된다. 무료 구간을 capacity로 선언하고
  정책 조건으로 삼는 것이 실체에 맞다.
- **비용 가중치 최적화** — 무료 구간 안에서 한계비용이 0이라 가중치
  곱이 퇴화한다. 용량 제약 캐시 계층이 같은 결과를 더 단순하게 낸다.
- **규칙 표현식 언어(DSL)** — 조건은 nullable 필드의 AND로 충분하다.
  새 조건은 파서가 아니라 필드 하나를 additive로 더한다.
