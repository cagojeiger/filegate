# ADR 001: provider는 코드가 아니라 설정이다

- Status: Draft
- Date: 2026-07-03
- 부모: [000](000-identity.md) 공리 1 (물리는 filegate가 소유한다)

## 문제

스토리지 벤더마다 기능, 주소, 인증 방식이 다르다. 벤더 상태도 변한다. 벤더를 코드에 고정하면 filegate도 그 변화에 묶인다. 파일 배치 지식은 서비스에 두지 않는다.

## 결정

- **provider는 접근 계약이다.** 벤더, 계정, 주소, 자격증명, 공간을 묶는다. 같은 벤더의 다른 계정도 다른 provider다.
- **S3 호환 API를 1차 계약으로 삼는다.** S3, R2, OCI, MinIO, Garage를 같은 adapter로 다룬다. 벤더 native adapter는 필요가 확인될 때만 추가한다.
- **내부 주소와 공개 주소를 구분한다.** 서명 URL은 주소에 묶인다. filegate가 쓰는 주소와 클라이언트가 쓰는 주소는 다를 수 있다.
- **capability는 선언식이다.** 런타임 탐지는 하지 않는다. 운영자가 선언하고, 틀리면 해당 provider 사용 시 실패한다.
- **file과 location을 분리한다.** file은 정체성이고, location은 바뀔 수 있는 위치다.
- **배치는 설정으로 고른다.** 대상은 후보 풀이고, 선택 전략은 설정이다. 기본값은 capacity 여유가 있는 후보 중 무작위다.

## 경계선

- 자동 비용 최적화와 이동은 요청 경로에 넣지 않는다. reconciler가 처리한다.
- 객체 이름은 filegate가 발급한 불투명 키다. 클라이언트 파일명이 아니다.
- 비용과 무료 구간은 [docs/vendors/](../vendors/README.md)에 둔다.

## 결과

- 벤더나 계정 추가는 설정 블록 추가로 처리한다. S3 호환이면 코드 변경은 없다.
- 로컬(MinIO)과 프로덕션(OCI)이 같은 adapter, 같은 코드 경로.
- 깨진 참조와 자격증명 누락은 부팅 실패로 처리한다.
